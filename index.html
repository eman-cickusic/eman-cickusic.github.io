Intelligent Cloud Operations Agent

LangChainPythonTerraformFastAPIGCP

Problem Statement

Cloud operations teams often rely on manual queries and console navigation for routine diagnostics and status checks. This project aimed to create an AI-powered agent to automate these tasks using natural language, providing instant Infrastructure-as-Code (IaC) awareness.

Architecture

Technical Decisions & Outcomes

  • Architectural Choice: A self-hosted Hugging Face model was chosen over proprietary APIs (like OpenAI) to ensure data privacy and eliminate API call costs, aligning with the project's zero-cost operational goal. The trade-off was increased management overhead of the GCE VM, which was acceptable for this proof-of-concept.
  • Technical Challenge: Securely parsing the Terraform state file from GCS was critical. This was solved by creating a dedicated GCP Service Account with only the `storage.objects.get` permission on the specific `.tfstate` object. The agent authenticates using this SA, adhering to the principle of least privilege.
  • Quantifiable Outcome: The agent successfully automated 5 core operational checks (CRUD in Firestore, log queries, BigQuery analysis, monitoring checks, and IaC lookups), reducing the time for these routine tasks from an average of 5-10 minutes manually to **under 15 seconds** via a natural language query.
