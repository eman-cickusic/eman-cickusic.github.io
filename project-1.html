Intelligent Cloud Operations Agent

LangChainPythonTerraformFastAPI

Problem Statement

Cloud operations teams rely on manual queries for routine diagnostics. This project aimed to create an AI agent to automate these tasks using natural language, providing instant Infrastructure-as-Code (IaC) awareness.

Architecture

Technical Decisions & Outcomes

  • Architectural Choice: A self-hosted Hugging Face model on a GCE VM was chosen over proprietary APIs to ensure data privacy and eliminate API costs, accepting the trade-off of increased management overhead.
  • Quantifiable Outcome: Automated 5 core operational checks, reducing the time for routine diagnostics from 5-10 minutes manually to under 15 seconds via natural language.

Security & Cost Optimization

  • Security Posture: Access to the Terraform state file was secured by creating a dedicated GCP Service Account with only the `storage.objects.get` permission on the specific `.tfstate` object, adhering to the principle of least privilege.
  • Cost Optimization: Using a self-hosted open-source LLM on a preemptible GCE instance significantly reduced operational costs compared to using a proprietary, per-token API like GPT-4, making the tool effectively free for internal use.

Testing Strategy

  • Unit Testing: The FastAPI backend logic, including the parsing of natural language queries, was validated using `pytest` to ensure predictable responses.
  • Integration Testing: A dedicated test script was created to verify the agent's connection and functional interaction with live GCP APIs (Logging, Firestore) in a non-production environment.
